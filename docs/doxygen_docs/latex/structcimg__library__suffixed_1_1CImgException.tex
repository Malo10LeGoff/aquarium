\doxysubsection{Detailed Description}
Instances of {\ttfamily \mbox{\hyperlink{structcimg__library__suffixed_1_1CImgException}{C\+Img\+Exception}}} are thrown when errors are encountered in a \textbackslash{}\+C\+Img function call. 

\begin{DoxyParagraph}{Overview}

\end{DoxyParagraph}
\mbox{\hyperlink{structcimg__library__suffixed_1_1CImgException}{C\+Img\+Exception}} is the base class of all exceptions thrown by \textbackslash{}\+C\+Img (except {\bfseries{\mbox{\hyperlink{structcimg__library__suffixed_1_1CImgAbortException}{C\+Img\+Abort\+Exception}}}}). \mbox{\hyperlink{structcimg__library__suffixed_1_1CImgException}{C\+Img\+Exception}} is never thrown itself. Derived classes that specify the type of errord are thrown instead. These classes can be\+:


\begin{DoxyItemize}
\item {\bfseries{\mbox{\hyperlink{structcimg__library__suffixed_1_1CImgAbortException}{C\+Img\+Abort\+Exception}}\+:}} Thrown when a computationally-\/intensive function is aborted by an external signal. This is the only {\ttfamily non-\/derived} exception class.
\item {\bfseries{\mbox{\hyperlink{structcimg__library__suffixed_1_1CImgArgumentException}{C\+Img\+Argument\+Exception}}\+:}} Thrown when one argument of a called \textbackslash{}\+C\+Img function is invalid. This is probably one of the most thrown exception by \textbackslash{}\+C\+Img. For instance, the following example throws a {\ttfamily \mbox{\hyperlink{structcimg__library__suffixed_1_1CImgArgumentException}{C\+Img\+Argument\+Exception}}\+:} 
\begin{DoxyCode}{0}
\DoxyCodeLine{CImg<float> \mbox{\hyperlink{namespacecimg__library__suffixed_a4b2be1eebcd69c87bde0b5757cad1938}{img}}(100,100,1,3); \textcolor{comment}{// Define a 100x100 color image with float-\/valued pixels}}
\DoxyCodeLine{\mbox{\hyperlink{namespacecimg__library__suffixed_a4b2be1eebcd69c87bde0b5757cad1938}{img}}.mirror(\textcolor{charliteral}{'e'});              \textcolor{comment}{// Try to mirror image along the (non-\/existing) 'e'-\/axis}}
\end{DoxyCode}

\item {\bfseries{\mbox{\hyperlink{structcimg__library__suffixed_1_1CImgDisplayException}{C\+Img\+Display\+Exception}}\+:}} Thrown when something went wrong during the display of images in \mbox{\hyperlink{classCImgDisplay}{C\+Img\+Display}} instances.
\item {\bfseries{\mbox{\hyperlink{structcimg__library__suffixed_1_1CImgInstanceException}{C\+Img\+Instance\+Exception}}\+:}} Thrown when an instance associated to a called \textbackslash{}\+C\+Img method does not fit the function requirements. For instance, the following example throws a {\ttfamily \mbox{\hyperlink{structcimg__library__suffixed_1_1CImgInstanceException}{C\+Img\+Instance\+Exception}}\+:} 
\begin{DoxyCode}{0}
\DoxyCodeLine{\textcolor{keyword}{const} CImg<float> \mbox{\hyperlink{namespacecimg__library__suffixed_a4b2be1eebcd69c87bde0b5757cad1938}{img}};           \textcolor{comment}{// Define an empty image}}
\DoxyCodeLine{\textcolor{keyword}{const} \textcolor{keywordtype}{float} value = \mbox{\hyperlink{namespacecimg__library__suffixed_a4b2be1eebcd69c87bde0b5757cad1938}{img}}.at(0);   \textcolor{comment}{// Try to read first pixel value (does not exist)}}
\end{DoxyCode}

\item {\bfseries{\mbox{\hyperlink{structcimg__library__suffixed_1_1CImgIOException}{C\+Img\+I\+O\+Exception}}\+:}} Thrown when an error occurred when trying to load or save image files. This happens when trying to read files that do not exist or with invalid formats. For instance, the following example throws a {\ttfamily \mbox{\hyperlink{structcimg__library__suffixed_1_1CImgIOException}{C\+Img\+I\+O\+Exception}}\+:} 
\begin{DoxyCode}{0}
\DoxyCodeLine{\textcolor{keyword}{const} CImg<float> \mbox{\hyperlink{namespacecimg__library__suffixed_a4b2be1eebcd69c87bde0b5757cad1938}{img}}(\textcolor{stringliteral}{"missing\_file.jpg"});  \textcolor{comment}{// Try to load a file that does not exist}}
\end{DoxyCode}

\item {\bfseries{\mbox{\hyperlink{structcimg__library__suffixed_1_1CImgWarningException}{C\+Img\+Warning\+Exception}}\+:}} Thrown only if configuration macro {\ttfamily cimg\+\_\+strict\+\_\+warnings} is set, and when a \textbackslash{}\+C\+Img function has to display a warning message (see \mbox{\hyperlink{namespacecimg__library__suffixed_1_1cimg_acfd3624d72ed7f79c82ec45646c685e7}{cimg\+::warn()}}).
\end{DoxyItemize}

It is not recommended to throw \mbox{\hyperlink{structcimg__library__suffixed_1_1CImgException}{C\+Img\+Exception}} instances by yourself, since they are expected to be thrown only by \textbackslash{}\+C\+Img. When an error occurs in a library function call, \textbackslash{}\+C\+Img may display error messages on the screen or on the standard output, depending on the current \textbackslash{}\+C\+Img exception mode. The \textbackslash{}\+C\+Img exception mode can be get and set by functions \mbox{\hyperlink{namespacecimg__library__suffixed_1_1cimg_a9414cac8b0ab20669320c37878fe54cc}{cimg\+::exception\+\_\+mode()}} and \mbox{\hyperlink{namespacecimg__library__suffixed_1_1cimg_aeafab144366eddd11360f32f468fa4af}{cimg\+::exception\+\_\+mode(unsigned int)}}.

\begin{DoxyParagraph}{Exceptions handling}

\end{DoxyParagraph}
In all cases, when an error occurs in \textbackslash{}\+C\+Img, an instance of the corresponding exception class is thrown. This may lead the program to break (this is the default behavior), but you can bypass this behavior by handling the exceptions by yourself, using a usual {\ttfamily try \{ ... \} catch () \{ ... \}} bloc, as in the following example\+:
